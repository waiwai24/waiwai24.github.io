# 基础概念

## 1.核心概念

传统的数据库（MySQL、PostgreSQL）用表格存储数据，像 Excel 表格一样。 图数据库（Neo4j 是目前最流行的图数据库）用节点 + 关系存储数据，像一张“关系网”。

| 概念                | 解释                                   | 举例                      |
| ------------------- | -------------------------------------- | ------------------------- |
| 节点 (Node)         | 代表实体，像表格里的一行记录           | 人、电影、城市            |
| 关系 (Relationship) | 节点之间的连接，有方向和类型           | 李雷 → KNOWS → 韩梅梅     |
| 属性 (Property)     | 节点或关系的键值对                     | 名字="李雷", 年龄=20      |
| 标签 (Label)        | 节点的分类（相当于表名）               | :Person, :Movie           |
| Cypher              | Neo4j 的查询语言（类似 SQL，但更直观） | MATCH (p:Person) RETURN p |



## 2.Cypher 语言

核心语法只有四个关键词：

- **MATCH**：找东西（像 SQL 的 SELECT）
- **CREATE**：创建东西
- **RETURN**：返回结果
- **WHERE**：加条件
